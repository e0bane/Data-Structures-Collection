1. Assignment Description

Let's assume you travel to Rome, Italy, and you plan to visit some famous, important world heritage sites, but you're short on time, so to make your itinerary work, you decide to find routes that connects all the important heritage sites which you want to visit with minimum total walking distance. For simplicity, let us assume that there are up to 50 heritage sites and maximum 100 routes between them.

You will be given a file containing the heritage sites and the distance information. See below for one such example:

Piazza Navona,Campo De Fiori(210.4),
Pantheon,Piazza Navona(300.5),Campo De Fiori(450.0),
End

Each line represents one heritage site and the route from this site to other site's distance. For above example,  line one indicates that there's one route from Piazza Navona to Campo De Fiori and the distance between them is 210.4 meters, where the second line indicates that there's one route from Pantheon to Piazza Navona and the distance is 300.5 meters, there's also a route from Pantheon to Campo De Fiori and the distance is 450.0 meters, etc.

Each input file will end with an "End".

2. Class description

Note: for each of the following classes, feel free to add your own auxillary functions in case it make your task easier.

Route

Inside Route class, the Site structure represents the vertex of the map.

struct Site
{
     string siteName;
     int rank;
     struct Site* parent;

};

Each Route object contains the following attributes.

 

Attribute's Name Data Type Descripiton
site1 and site2 Site* This represents the two vertice that connects the Route
distance double This represents the length/weight of the Route between site1 and site2

 

Functions in Route class are defined as below:

 

Function Descipriton
Route(); C++ doesn't provide a default contructor, use this one to initialize all instance variables
Route(Site* site1, Site* site2, double distance); This is the overloaded constructor.
Site* getSite1(); Accessor for instance variable site1
Site* getSite2(); Accessor for instance variable site2
double getDistance(); Accessor for instance variable distance
void printRoute(); This function prints the Route info. on screen

Map

The Map class represents an undirected graph. Each Map object contains the following attributes.

Attribute's Name Data Type Descripiton
numOfSites or numOfRoutes int numOfSites and numOfRoutes represents the actual number of Site (vertice) and Route (edges) inside the graph.
setOfSites Site* This represents a dynamic array containing Site objects. The maximum size of the array should be 50.
setOfRoutes Route* This represents a dynamic array containing Route objects. the maximum size of the array should be 100.

 

Functions in Map class are defined as below:

 

Function Descipriton
Map(int numOfSites, int numOfRoutes, Site* siteArray, Route* routeArray); Contructor. Used to initialize all instance variables
~Map(); Destructor. Use to release all dynamic allocated memory.
void make_set(Site* aSite); Given a Site object, this function creates a new set whose only member is
the Site itself.
Site* find_set(Site* aSite); Given a Site object, this function returns a pointer that points to the representative
of the (unique) set containing 'aSite'
void link(Site* aSite, Site* bSite); Given two Site objects, this function checks the dynamic sets that contain the parameterized
'aSite' and 'bSite' and update their rank and parent info. accordingly
void Union(Site* aSite, Site* bSite); Given two Site objects, this function unites the dynamic sets that contain the parameterized
'aSite' and 'bSite' into a new set that is the union of the two sets.
void sortRoutes(); This function sorts all Routes according to their length in non-decreasing order. Note: use the best sorting algorithms, such as MergeSort, QuickSort or HeapSort. You are NOT allowed to use insertion sort, selection sort, or Bubble sort, etc.
void MST_Kruskal(); This function performs the Kruskal algorithm on the graph.

*Important: you are NOT allowed to change above class Map definition, i.e. you are not allowed to change attribute setOfSites data type from Site* to Site**, or change attribute setOfRoutes from Route* to Route**, etc.

3. Misce

1) In Kruskal's algorithm, you will need to sort all Routes in non-decreasing order according to their weight. In this assignment, you're required to use either MergeSort, QuickSort or HeapSort to do this. You can NOT use inefficient sorting algorithms, such as insertion sort, selection sort, or bubble sort, etc.

2) In this assignment, you will need to print all Routes with their distance in the order that they were added to the MST, and also print the total distance. Note: in the Kruskal's algorithm, when an Route is added to the set A, you have the following union operation:

A = A U { ( u, v ) }

In this programming assignment, you just need to print each Route (with their vertex names and its weight) instead of actually adding it to set A, so that your Kruskal function does not need to return the set A. See below for one sample output.

Total historical sites you will visit: 3

MST Kruskal Algorithm Result

Pantheon              <--->       Piazza Navona     = 300.5
===========================================================
Total Distance: 510.9 meters
